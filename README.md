## 🛠 Как запускать проекты

1. Клонируй репозиторий:
   ```bash
   git clone https://github.com/MoZiLLa00/js-learning.git
   cd js-learning
   ```

Блок 1: Переменные, типы данных, ввод/вывод
🛠 Основные элементы
Элемент / Описание
prompt() / alert() / console.log() / Ввод данных от пользователя, вывод уведомлений и отладочная печать
let / const / Объявление изменяемых и неизменяемых переменных
Number() / Number.isNaN() / Явное приведение строк к числам и проверка на корректность
new Date().getFullYear() / Динамическое получение текущего года

🧪 Запуск

1. Открой block-1/age-calculator/index.html в браузере
2. Введи год рождения в появившееся окно
3. Результат (возраст или сообщение об ошибке) отобразится в alert и консоли разработчика

💡 Ключевые концепции

- Строгая типизация при вводе: prompt всегда возвращает строку, которую нужно преобразовать
- Валидация данных: проверка на null, пустую строку и NaN перед вычислениями
- Использование const по умолчанию, let только при необходимости изменения
- Чтение ошибок в консоли (TypeError, ReferenceError) для быстрой отладки

Блок 2: Операторы и ветвление
🛠 Логика и ветвления
Конструкция / Описание
if / else if / else / Последовательная проверка условий и выбор ветки выполнения
switch / case / Выбор действия из множества вариантов на основе одного значения
Тернарный оператор ? : / Короткая запись условия для присваивания значений
Логические операторы &&, `

🧪 Запуск

1. Открой block-2/quiz-app/index.html в браузере
2. Последовательно ответь на 3 вопроса (столица, математика, логика)
3. Получи итоговый результат и сообщение в зависимости от количества правильных ответов

💡 Ключевые концепции

- Строгое сравнение === / !== вместо неявного приведения типов ==
- Truthy/Falsy значения: как JS интерпретирует строки, числа и объекты в условиях
- Использование switch для дискретных значений и if для диапазонов или сложной логики
- Подсчёт баллов через счётчик и ветвление финального вывода

Блок 3: Циклы
🛠 Типы циклов и паттерны
Цикл (Метод) / Описание
for / Классический счётчик для известного количества повторений (лесенка)
while + continue /Цикл с условием выхода и пропуском итераций (обратный отсчёт)
Вложенные for / Генерация двумерных структур (таблица умножения)
.repeat(n) / Дублирование символов без ручного конкатенирования

🧪 Запуск

1. Открой block-3/pattern-generator/index.html в браузере
2. Выбери номер паттерна (1–4) и введи параметры (размер, число для пропуска)
3. Увидь сгенерированный текстовый рисунок в консоли и всплывающем окне

💡 Ключевые концепции

- Разница между while (проверка до выполнения) и do...while (минимум 1 выполнение)
- Управление потоком: break для выхода, continue для пропуска шага
- Предотвращение бесконечных циклов: гарантированное изменение счётчика внутри тела
- Сборка строк через конкатенацию перед единовременным выводом для форматирования

Блок 4: Функции
🛠 Библиотека функций
Функция / Описание
add(a, b) / Сложение (Function Declaration)

🧪 Запуск

1. Открой block-4/calculator-lib/index.html в браузере
2. Выбери математическую операцию и введи числа
3. Результат отобразится в alert, а автоматические тесты — в консоли

💡 Ключевые концепции

- Три способа объявления функций и правила их использования
- Чистые функции и побочные эффекты: предсказуемость и лёгкость тестирования
- Колбэки: передача логики как аргумента для гибкости и переиспользования
- Параметры по умолчанию и rest-оператор ... для работы с переменным числом аргументов
- Базовое тестирование через console.assert для быстрой проверки корректности

Блок 5: Объекты и массивы — Task Manager

### 🛠 Функции

| Функция | Описание

| `addTask(tasks, title, priority)` | Добавляет задачу с авто-генерацией ID |
| `toggleTask(tasks, id)` | Переключает статус `done` (иммутабельно) |
| `deleteTask(tasks, id)` | Удаляет задачу по ID |
| `getTasksByStatus(tasks, status)` | Фильтр: `"done"` / `"pending"` |
| `getTasksByPriority(tasks, priority)` | Фильтр по приоритету |
| `getStats(tasks)` | Возвращает статистику: `{ total, done, pending, highPriority }` |

### 🧪 Запуск

1. Открой `index.html` в браузере
2. Выбери операцию в меню
3. Результат отобразится в консоли и `alert`

### 💡 Ключевые концепции

- Иммутабельность: функции возвращают новые массивы, не меняя оригинал
- Методы массивов: `map`, `filter`, `reduce`
- Деструктуризация и spread-оператор `...`
- Тернарные операторы для чистого кода
